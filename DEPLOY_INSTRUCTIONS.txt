三、【绝对禁止】操作清单
------------------------
以下操作严格禁止执行，违者可能导致数据丢失或服务崩溃：

  禁止1: 删除服务器上的任何目录或文件

  禁止9: 修改 PM2 配置文件（除非部署包内包含新版本）

五、部署步骤（严格按顺序执行）
------------------------------
步骤1: 上传部署包到服务器临时目录
  上传文件: trigen-deploy-safe.tar.gz
  上传路径: /tmp/trigen-deploy-safe.tar.gz

步骤2: 备份白名单内的旧文件（可选，建议执行）
  只备份即将被覆盖的文件，备份到 /tmp/trigen_backup_日期/
  备份内容仅限白名单内的文件

步骤3: 解压部署包到临时目录
  mkdir -p /tmp/trigen-deploy/
  tar -xzvf /tmp/trigen-deploy-safe.tar.gz -C /tmp/trigen-deploy/

步骤4: 按白名单逐个复制文件
  【重要】只复制白名单中列出的文件，禁止 cp -r 整个目录！
  
  复制方式：
  - 对于白名单中已存在的文件：覆盖更新
  - 对于白名单中新文件：新增
  - 对于白名单外的文件：一律不动

步骤5: 安装npm依赖
  cd /home/admin/he-wiki-rag/
  npm install --legacy-peer-deps
  【注意】只安装/更新依赖，不删除任何现有包

步骤6: 重启PM2服务
  pm2 restart trigen
  【注意】使用 restart，不要使用 delete + start

步骤7: 验证服务状态
  pm2 status
  检查应用是否为 online 状态

步骤8: 清理临时文件
  rm -rf /tmp/trigen-deploy/
  rm -f /tmp/trigen-deploy-safe.tar.gz
  【注意】只清理临时目录，不动部署目录

-----------
如果部署出现问题，按以下步骤回滚：

1. 停止服务：pm2 stop trigen
2. 从备份目录恢复白名单文件
3. 重启服务：pm2 start trigen
4. 验证服务恢复正常
