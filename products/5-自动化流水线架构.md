## 第一步：配置 GitHub Actions（一次设置，永久生效）

### 添加 GitHub Secrets

打开 GitHub 仓库 → Settings → Secrets and variables → Actions

添加以下 3 个 secrets：

| Secret 名称 | 值 | 说明 |
|-------------|-----|------|
| `SERVER_HOST` | `120.79.17.184` | 阿里云服务器 IP |
| `SERVER_USER` | `root` | SSH 用户名 |
| `SERVER_SSH_KEY` | 粘贴 `id_ed25519` 私钥内容 | SSH 私钥 |

### 获取 SSH 私钥内容
在本地终端执行：
```bash
cat ~/.ssh/id_ed25519
```
复制全部内容粘贴到 `SERVER_SSH_KEY`

### 启用 Actions
- GitHub 会自动检测 `.github/workflows/deploy.yml`
- 下次 push 到 main 就会自动触发部署
- 也可手动触发：仓库 → Actions → Auto Deploy → Run workflow

## 第三步：标准工作流程

### 日常改代码（Trae → 自动部署）
```
1. Trae 中修改 index.html / server.js
2. git add . && git commit -m "xxx"
3. git push
4. ✅ GitHub Actions 自动部署到阿里云
5. ✅ 网站自动更新 http://120.79.17.184:3001
```

### 新需求处理（WorkBuddy → Trae）
```
1. 跟我说需求
2. 我出方案/内容/指令文件
3. 你拿去 Trae 执行代码修改
4. Trae push → 自动上线
5. 我每日检查确认状态
```

### 日常运维（WorkBuddy 自动化）
```
每天 20:00 自动检查：
• 网站是否在线
• 面包多商品状态
• 待办提醒
```

## 故障处理

| 问题 | 解决 |
|------|------|
| 部署失败 | 检查 GitHub Secrets 是否配置正确 |
| SSH 连不上 | 检查 120.79.17.184:22 是否放行 |
| PM2 重启失败 | SSH 登录后 `pm2 status` 查看具体错误 |
