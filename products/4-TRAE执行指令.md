# 🧐 Trae 执行指令：代码审查 + 网站查漏补缺

---

## ⚠️ 你的核心使命

**你只做三件事：**
1. **代码审查** — 检查整个项目代码质量、安全隐患、性能问题
2. **查漏补缺** — 发现功能缺失、UI 异常、逻辑漏洞
3. **完善质量** — 修复发现的问题，提升网站品质

**其他所有事项（部署、运维、API 接入、支付、合规）→ 交给 WorkBuddy，不要碰。**

---

## 📋 例行审查清单

每次执行时，按以下顺序走一遍：

### 1️⃣ 前端质量检查
- [ ] HTML 结构是否完整（DOCTYPE、meta、语义化标签）
- [ ] CSS/样式是否正常（响应式、兼容性、加载速度）
- [ ] JavaScript 是否有报错（打开浏览器控制台）
- [ ] 所有链接是否可点击（导航、按钮、外部链接）
- [ ] 图片是否正常加载、alt 属性是否齐全
- [ ] 移动端适配是否正常

### 2️⃣ 功能验证
- [ ] 聊天功能：输入 → 发送 → 回复是否正常
- [ ] 模型切换：切换不同 API 模型是否正常响应
- [ ] 小说版块：页面加载、内容展示是否正确
- [ ] Agent 模块：交互流程是否完整
- [ ] 画廊模块：图片展示、加载是否正常
- [ ] 支付入口：面包多链接是否正确跳转
- [ ] 积分系统：页面显示是否正常

### 3️⃣ 安全隐患
- [ ] 有没有 API Key 明文暴露在前端代码中
- [ ] 有没有敏感信息出现在控制台日志
- [ ] 外部请求是否通过安全通道（HTTPS）
- [ ] 用户输入有没有 XSS 防护

### 4️⃣ 性能优化
- [ ] 是否有加载缓慢的模块
- [ ] 是否有多余的 console.log 未清理
- [ ] 图片/资源是否需要压缩或懒加载
- [ ] 是否有未使用的 CSS/JS 需要清理

## 🔧 问题处理流程

| 问题类型 | 处理方式 |
|----------|----------|
| 简单修复（改几行代码） | 直接改，然后告知我改了什么 |
| 中等修复（需重构模块） | 先在文件内写 TODO 注释，口头汇报给我 |
| 重大改动（架构调整） | **不要动**，口头汇报给我，我来决定 |
